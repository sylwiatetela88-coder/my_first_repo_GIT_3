Notatka z ćwiczenia GIT_& 
Poleceni git branch tworzy <nazwa> tworzy gałąź, git checkout <nazwa> przełącza na gałąź, git checkout -b <nazwa> tworzy i przełącza na tą gałąź. Każda z gałęzi pozwala niezależnie rozwijąć prjekt główny. Możemy sobie testować różne zmiany bez wpływu na głóną gałęź. Polecenie merge pozwala scalić wszystkie gałęzie , jak gałęzie zostaną zmienione niezależnie od siebie a później zostaną scalone powstaje konflikt. Konflikt można usunąć ręcznie przy pomocy usuniecia znaczników "<,>,=". Po roziązaniu konflitu należy wykonać git add, dodać git commit -m o rozwiązaniu konfliktu . git log --oneline --grph pokazuje jak historia gałęzi wyglada graficznie.
